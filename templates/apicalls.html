API calls list

Authentication

  • Authenticate with POST /api/auth/login (JSON or form body: username, password). Returns a JWT token valid for 24 hours.
  • Send it on every subsequent call as an HTTP header: Authorization: Bearer <token>.
  • If you're logged in on this site, you can also get a fresh token from the dashboard ("Generate" button) without re-entering your password.

Rate limits

  • Anonymous (no Authorization header): 2 /api/forecast GET requests per day, tracked by IP address.
  • Free (registered user with a token): 100 /api/forecast GET requests per day, tracked by user id.
  • Premium (registered user with a token and premium plan): unlimited /api/forecast GET requests, plus every query is automatically saved to your query history (see /api/queries below).
Url Methods Description Parameters
/api/auth/login POST Authenticates with username + password and returns a JWT access token.
  • username - Account username (required)
  • password - Account password (required)
/api/forecast GET, PUT, DELETE Manages forecast data. GET works with or without a token (see rate limits above). PUT is an upsert (creates if it doesn't exist yet for that placeid+date, otherwise updates it) and always requires a valid token ONLY WORKS WITH ADMIN TOKEN, same as DELETE.
    GET
  • Authorization header optional (enables the free/premium tier and, for premium users, saves the query to history)
  • placename - Place name
  • placeid - Place ID
  • date - Date in ISO format (optional, defaults to today)
  • time - Time of day, format 'HH:MM' (optional)
  • details - Whether to include additional details (condition description and place name) (optional)
    PUT
  • Authorization header required
  • placeid - Place ID
  • date - Date in ISO format (required)
  • condition - Weather condition id (optional)
  • rain - Rain (mm) (optional)
  • humidity - Humidity percentage (optional)
  • wind - Wind speed (km/h) (optional)
  • wind_direction - Wind direction (optional)
    DELETE
  • Authorization header required
  • forecastid - Forecast id
/api/places GET, PUT, DELETE Retrieves place data (any authenticated user). Creating/updating/deleting places requires an admin account.
  • Authorization header required for all methods
    GET
  • placeid - Place id
  • placename - Place name
    PUT (admin only)
  • placeid - Place id
  • placename - Place name
  • lat - Latitude (optional)
  • lon - Longitude (optional)
  • coords - Coordinates 'float,float' (optional)
    DELETE (admin only)
  • placeid - Place id
  • placename - Place name
/api/conditions GET Retrieves weather condition data: supports querying by condition ID
  • Authorization header required
    GET
  • id - Condition ID (optional)
/api/queries GET Premium only. Lists the authenticated user's saved forecast query history (JSON, most recent first).
  • Authorization header required (premium account)
/api/queries/<id> GET Premium only. Returns the full detail of one saved query, including the original forecast result (JSON).
  • Authorization header required (premium account)

Examples

  • curl -X POST http://localhost:5000/api/auth/login -H "Content-Type: application/json" -d "{\"username\":\"premium_demo\",\"password\":\"premium12345\"}"
  • curl "http://localhost:5000/api/forecast?placeid=1&date=2026-07-01" (anonymous, no token needed, only 2 request per day)
  • curl -H "Authorization: Bearer <TOKEN>" "http://localhost:5000/api/forecast?placename=Bruxelles&date=2026-07-01"
  • curl -H "Authorization: Bearer <TOKEN>" "http://localhost:5000/api/queries" (premium account)
